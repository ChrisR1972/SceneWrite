PURPOSE AND OVERVIEW
================================================================================

The Screenplay Writing Tool is a professional desktop application (PyQt6-based)
designed to help creators generate and manage screenplay storyboards optimized
specifically for higgsfield.ai video generation. The app bridges the gap between
story ideation and AI video production by providing a structured workflow from
premise to detailed video prompts.

The application is built in Python 3.10+ using PyQt6 for the GUI and supports
both cloud-based AI providers (OpenAI, Together AI, OpenRouter, Hugging Face)
and local AI servers (Ollama, LM Studio).

DATA MODEL HIERARCHY
================================================================================

The application uses a hierarchical data structure:

Screenplay (Top Level)
  ├── Metadata: title, premise, genre, atmosphere, story_structure
  ├── Acts (List of StoryAct)
  │   ├── Act metadata: act_number, title, description, plot_points, character_arcs
  │   └── Scenes (List of StoryScene)
  │       ├── Scene metadata: scene_id, scene_number, title, description
  │       ├── Scene details: plot_point, character_focus, pacing, estimated_duration
  │       └── Storyboard Items (List of StoryboardItem)
  │           ├── Item metadata: item_id, sequence_number, duration (1-30 seconds, AI-optimized)
  │           ├── Prompts: image_prompt (static first frame), prompt (video action)
  │           ├── Content: visual_description, dialogue, camera_notes
  │           ├── Character references: identity_block (visual consistency)
  │           └── Scene type: ACTION, DIALOGUE, TRANSITION, etc.
  └── Storyboard Items (flat list, all items across all scenes)
      └── Same structure as above

Key Data Classes:
- Screenplay: Container for entire story with acts, scenes, and storyboard items
- StoryAct: Represents a story act (Act 1, Act 2, etc.) with scenes
- StoryScene: Represents a scene within an act with storyboard items
- StoryboardItem: Individual video segment (1-30 seconds, AI-optimized) with prompts and metadata
- StoryOutline: Contains main_storyline, subplots, characters, conclusion


COMPLETE WORKFLOW
================================================================================

WORKFLOW 1: AI-GENERATED STORY (Story Creation Wizard)
------------------------------------------------------

Step 1: Premise Creation (premise_step.py)
  - User selects genres (checkboxes: Action, Drama, Horror, etc.)
  - User selects atmosphere/tone (dropdown: Suspenseful, Dark, Lighthearted, etc.)
  - User selects story length:
    * Micro: 1 act, 1-5 scenes (for ads, loops, single-scene narratives)
    * Short: 3 acts, 9-15 scenes (social media, trailers, proof-of-concept)
    * Medium: 3 acts, 15-24 scenes (YouTube shorts, episodic pilots)
    * Long: 5 acts, 30-50 scenes (feature-length planning, episodic breakdowns)
  - User sets character count (2-10 characters)
  - User optionally enters a title
  - Click "Generate Premise" → AI generates a story premise
  - User can edit the generated premise
  - Click "Next" to proceed

Step 2: Story Outline Generation (story_outline_step.py)
  - AI generates comprehensive story outline including:
    * Main storyline (detailed narrative arc)
    * Subplots (2-4 secondary storylines, 2-3 sentences each)
    * Characters (name, outline, growth arc for each)
    * Conclusion (how the story resolves)
  - User can edit any section
  - User can regenerate individual sections (subplots, conclusion, characters)
  - Click "Next" to proceed

Step 3: Framework Generation (framework_generation_step.py)
  - AI generates complete story framework:
    * Creates acts based on selected length
    * Creates scenes within each act (based on length specifications)
    * For each scene, generates:
      - Title and description (3-5 sentences)
      - Plot point (Inciting Incident, Midpoint, Climax, etc.)
      - Character focus (which characters appear)
      - Pacing (Fast/Medium/Slow)
      - Estimated duration (30-180 seconds)
  - Framework is displayed in a tree structure
  - Click "Finish" to complete wizard

Result: A complete Screenplay object with acts, scenes, and framework structure
(but no detailed storyboard items yet - those are generated separately)


WORKFLOW 2: MANUAL STORY CREATION
---------------------------------

Option A: Manual Story Dialog
  - File → New Story (Manual)
  - Enter title and optional premise
  - Creates empty Screenplay object

Option B: Direct Premise Entry
  - Use PremiseDialog directly
  - Enter premise manually or generate with AI
  - Creates Screenplay with premise only


WORKFLOW 3: STORYBOARD GENERATION (Phase 2)
--------------------------------------------

After framework is created, user can generate detailed storyboard items:

1. User selects a scene (or all scenes) in the Story Framework View
2. Click "Generate Storyboard" or "Generate Storyboard for Scene"
3. AI generates detailed storyboard items for each scene:
   - Breaks down each scene into storyboard items with AI-optimized durations
   - Creates image_prompt: Static first frame description (for higgsfield.ai image generation)
   - Creates prompt: Video action description with explicit action verbs
   - Generates visual_description, dialogue, camera_notes
   - Assigns optimal duration per item (1-30 seconds based on content and pacing)
   - Creates sequence numbers for ordering

Key Distinction:
- Image Prompt: Describes static moment, no action verbs, for first frame
- Video Prompt: Describes actions with explicit verbs, for video generation
- Both must describe the SAME setting/environment for consistency

4. Storyboard items appear in the Storyboard Timeline view
5. User can click any item to edit in StoryboardItemEditor


WORKFLOW 4: EDITING AND REFINEMENT
-----------------------------------

Framework Editing (story_framework_view.py):
  - Tree view shows: Acts → Scenes → Storyboard Items
  - User can edit act/scene titles, descriptions, plot points
  - User can add/remove scenes
  - User can reorder scenes via drag-and-drop
  - Premise tab: Edit premise, subplots, conclusion
  - Characters tab: Edit character details, outlines, growth arcs
  - Identity Blocks tab: Manage character visual consistency blocks

Storyboard Item Editing (storyboard_item_editor.py):
  - Edit duration (1-30 seconds, AI-optimized default)
  - Edit image_prompt (static first frame)
  - Edit prompt (video action)
  - Edit visual_description, dialogue, camera_notes
  - Regenerate prompts with AI assistance
  - Link to character identity blocks for visual consistency
  - Set scene type (ACTION, DIALOGUE, TRANSITION, etc.)

AI Chat Panel (ai_chat_panel.py):
  - Context-aware chat assistant
  - Can answer questions about the screenplay
  - Can suggest improvements
  - Can help with prompt refinement


WORKFLOW 5: EXPORT
------------------

Export Formats (higgsfield_exporter.py):
  1. JSON Export: Complete screenplay data in JSON format
     - Includes all metadata, acts, scenes, storyboard items
     - Optimized for higgsfield.ai import
  2. CSV Export: Spreadsheet-compatible format
     - One row per storyboard item
     - Columns: Sequence, Duration, Image Prompt, Video Prompt, etc.
  3. Prompts Only: Simple text file
     - One prompt per line
     - Image prompt and video prompt separated

Export Process:
  - File → Export → Select format
  - Choose save location
  - File is generated with all storyboard data

TECHNICAL DETAILS
================================================================================

AI PROMPT ENGINEERING
---------------------

Premise Generation:
  - Uses structured prompts with genre, atmosphere, and length constraints
  - Returns raw AI output for transparency
  - Extracts clean premise from AI response

Story Outline Generation:
  - Generates main storyline, subplots, characters, and conclusion
  - Validates that all fields are populated
  - Handles both string and array formats for subplots
  - Can regenerate individual sections

Framework Generation:
  - Creates acts and scenes based on length specifications
  - Generates scene descriptions, plot points, character focus, pacing
  - Estimates duration for each scene (30-180 seconds)
  - Returns structured JSON with acts and scenes

Storyboard Generation:
  - Breaks scenes into segments with AI-optimized durations
  - Creates separate image_prompt (static) and prompt (action)
  - Ensures prompts describe same setting/environment
  - Generates dialogue, camera notes, visual descriptions
  - Determines duration based on scene complexity

PROMPT STRUCTURE FOR HIGGSFIELD.AI:
  - Image Prompt: Static description, no action verbs, first frame only
  - Video Prompt: Action description with explicit verbs, same setting
  - Both must match in: location, environment, lighting, time of day

INTEGRATION WITH HIGGSFIELD.AI
================================================================================

The app is specifically designed to generate prompts optimized for higgsfield.ai:

1. Image Prompts:
   - Static first frame descriptions
   - No action verbs
   - Detailed visual information
   - Used for generating the starting image

2. Video Prompts:
   - Action descriptions with explicit verbs
   - Same setting/environment as image prompt
   - Detailed movement and action
   - Used for generating video from the image

3. Export Format:
   - JSON structure with image_prompt and prompt fields
   - Sequence numbers for ordering
   - Duration information
   - Character identity blocks for consistency

4. Character Consistency:
   - Identity blocks store character appearance descriptions
   - Linked to storyboard items
   - Ensures consistent character rendering across scenes
