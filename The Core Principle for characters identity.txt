The Core Principle

A character’s physical identity must be immutable.
Clothing and props are scene-level state.

If MoviePrompterAI treats clothes as part of the character identity, consistency will always drift.

So the solution is to split character identity into two layers:

1. Character Identity (Global / Persistent)

This never changes across the story.

Stored once and reused everywhere.

Includes:

Face shape

Hair color & style

Eye color

Skin tone

Age range

Build / body type

Distinctive features (scar, tattoo, glasses if permanent)

This is what you use to generate the canonical reference image.

Example — Character Identity Block
CHARACTER: HENRY KANE

physical_identity:
- Male, mid-30s
- Angular face, sharp jawline
- Short dark hair, slightly tousled
- Brown eyes
- Light stubble
- Lean athletic build
- Small scar above left eyebrow


👉 This generates one master reference image.

2. Character Wardrobe (Scene-Level / Mutable)

This changes per scene and must never overwrite identity.

Includes:

Clothing

Accessories

Armor

Uniforms

Weather-appropriate layers

Damage / dirt / blood (if relevant)

This lives in the scene identity block, not the character definition.

Example — Scene Character State
CHARACTER STATE: HENRY KANE

wardrobe:
- Dark leather jacket
- Black t-shirt
- Faded jeans
- Heavy boots

3. Image Generation Strategy (This is the key)

You were absolutely right in your intuition.

Best practice for image generation:

Always pass the canonical character image as reference

Never regenerate the face

In the prompt, explicitly state:

“Same character as reference image”

“Different clothing as described below”

Example — Image Generation Prompt (Per Scene)
Use the provided reference image for HENRY KANE to maintain facial identity.

The character is wearing:
- Dark leather jacket
- Black t-shirt
- Faded jeans
- Heavy boots

Maintain identical face, hair, and body proportions.


This works well with:

ControlNet

IP-Adapter

Reference-image conditioning

Most modern video AIs

4. What NOT to Do ❌

❌ Do NOT include clothes in the base character identity

❌ Do NOT regenerate a new “full body” character per scene

❌ Do NOT allow wardrobe to leak into the global character block

Those are the #1 causes of visual drift.

🔧 CURSOR PROMPT — CHARACTER IDENTITY VS WARDROBE

Paste this into Cursor:

MoviePrompterAI must strictly separate character identity from scene-level appearance.

---

1. CHARACTER IDENTITY RULE (GLOBAL)

Character identity includes:
- Face structure
- Hair color and style
- Eye color
- Skin tone
- Age range
- Body build
- Permanent features (scars, tattoos, glasses if permanent)

Character identity is:
- Generated once
- Immutable across the story
- Used to create a single canonical reference image

Clothing MUST NOT appear in the character identity block.

---

2. CHARACTER WARDROBE RULE (SCENE-LEVEL)

Clothing, accessories, and temporary appearance details:
- Are scene-specific
- Must be stored in the scene identity block
- Must never overwrite character identity

Examples:
- Clothing
- Armor
- Uniforms
- Dirt, blood, damage
- Weather layers

---

3. IMAGE GENERATION RULE

When generating images or video:
- Always use the canonical character reference image
- Never regenerate the character’s face or body
- Explicitly instruct the model to preserve identity

Wardrobe changes must be described separately.

---

4. VALIDATION PASS (REQUIRED)

Before generating a scene:
- Verify no clothing appears in the character identity block
- Verify wardrobe exists only at scene level
- Reject any prompt that mixes identity and wardrobe

---

- Stable character appearance across all scenes
- Clean wardrobe variation
