We are enforcing a strict Action Verb Whitelist for SceneWrite.

Action markup (* *):
- Represents visible, camera-capturable motion or state change
- Must use ONLY approved verbs
- Must NOT express internal thoughts or emotions

---

1. ACTION MARKUP RULE (MANDATORY)

Any text inside *asterisks* MUST:
- Be a single action verb or verb phrase
- Exist in the approved Action Verb Whitelist
- Describe something visually observable

If a verb is not on the whitelist:
→ It MUST NOT be used as action markup.

---

2. ENTITY RELATIONSHIP RULE

Actions:
- Animate existing entities
- Do NOT create characters, objects, vehicles, or environments

No identity extraction may be triggered by actions alone.

---

3. REWRITE RULE

If a sentence contains:
- Internal emotion
- Mental state
- Abstract description

Rewrite it into:
- Physical action
- Environmental response
- Or remove it entirely

---

4. VALIDATION PASS (REQUIRED)

After scene generation:
- Scan all *action* markup
- Verify every action verb exists in the whitelist
- If not:
  - Replace with closest valid action
  - Or remove markup

---

EXPECTED RESULT:

- All actions are explicit and visual
- Motion prompts are reliable
- No emotional or abstract leakage
- Scene content behaves like a screenplay, not prose

How Action Markup Must Be Used

Format:

*action verb*

Rules:

One action per markup
Must come from the whitelist
Must be visually observable
Must apply to an existing entity

Example (Correct)
The {Stellar Wanderer} begins to *tremble*.
HENRY *grabs* the [console] and *braces*.
Lights *flicker* across _Cockpit_.

Example (Incorrect)
HENRY *realizes* something is wrong.
The ship *feels* unstable.
